# MediLabs

MediLabs adalah aplikasi web Sistem Reservasi dan Pendaftaran Laboratorium Klinik yang dikembangkan untuk project mata kuliah Rekayasa Perangkat Lunak. Aplikasi ini dirancang untuk membantu pasien dalam melakukan reservasi pemeriksaan laboratorium secara online, melihat daftar layanan pemeriksaan, mengecek status reservasi, serta membantu admin dalam mengelola data reservasi pasien.

Project ini dikembangkan secara bertahap mulai dari perancangan UI, implementasi front-end, hingga pengembangan fitur lanjutan. Pada tahap akhir atau 100%, sistem MediLabs ditargetkan memiliki halaman pengguna dan halaman admin agar proses reservasi dapat berjalan lebih terstruktur.

## Tujuan Project

Tujuan utama dari pengembangan MediLabs adalah menyediakan sistem berbasis web yang memudahkan proses pendaftaran dan reservasi pemeriksaan laboratorium klinik. Dengan adanya sistem ini, pasien dapat melihat layanan yang tersedia, membuat reservasi, memperoleh kode reservasi, dan mengecek status pemeriksaan secara online.

Selain itu, sistem ini juga membantu pihak laboratorium dalam memantau reservasi yang masuk melalui halaman admin. Admin dapat melihat ringkasan reservasi, mengecek detail data pasien, serta mengelola status reservasi.

## Tahapan Pengembangan

### Tahap 25%

Pada tahap 25%, pengembangan difokuskan pada pembuatan tampilan awal aplikasi. Halaman yang dibuat meliputi landing page, form reservasi, halaman hasil reservasi, serta halaman cek status sederhana. Data yang digunakan masih berupa data dummy dan belum terhubung ke database.

### Tahap 50%

Pada tahap 50%, tampilan aplikasi mulai dilengkapi dengan halaman layanan, detail layanan, riwayat reservasi, dan alur reservasi yang lebih jelas. Pada tahap ini, pengguna sudah dapat memahami alur dasar penggunaan aplikasi mulai dari melihat layanan hingga melakukan reservasi.

### Tahap 100%

Pada tahap 100%, sistem dikembangkan dengan fitur yang lebih lengkap, termasuk halaman admin. Admin dapat melihat dashboard, mengecek status reservasi, dan mengelola data reservasi pasien. Tahap ini menjadi gambaran akhir dari sistem MediLabs sebagai aplikasi reservasi dan pendaftaran laboratorium klinik berbasis web.

## Struktur Umum Sistem

MediLabs memiliki dua jenis pengguna utama, yaitu pasien dan admin. Pasien menggunakan sistem untuk melihat layanan, membuat reservasi, dan mengecek status pemeriksaan. Admin menggunakan sistem untuk memantau dan mengelola reservasi yang masuk.

Alur sistem secara umum adalah sebagai berikut:

1. Pasien membuka landing page MediLabs.
2. Pasien melihat daftar layanan pemeriksaan.
3. Pasien membuka detail layanan.
4. Pasien mengisi data pasien dan form reservasi.
5. Sistem menampilkan hasil reservasi berupa kode reservasi.
6. Pasien dapat mengecek status reservasi.
7. Admin dapat melihat dan mengelola reservasi melalui dashboard admin.

## Status Project

Project ini masih dikembangkan sebagai bagian dari pembelajaran Rekayasa Perangkat Lunak. Fokus utama project adalah memahami proses pembangunan perangkat lunak mulai dari analisis kebutuhan, desain sistem, desain antarmuka, implementasi front-end, hingga persiapan pengembangan backend dan database.

Pada tahap saat ini, sebagian fitur masih menggunakan data dummy. Pengembangan selanjutnya dapat diarahkan pada integrasi database, autentikasi pengguna, validasi backend, CRUD reservasi, dan pengelolaan status reservasi secara dinamis.
